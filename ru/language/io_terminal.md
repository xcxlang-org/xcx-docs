# XCX 3.1 — ввод-вывод и системные команды

## Вывод в консоль (`>!`)

Оператор `>!` выводит значения в `stdout` с переводом строки в конце.

```xcx
>! "Hello";
>! 42;
>! "Path: " + path;
```

Поддерживаются **escape-последовательности** `\n`, `\t`, `\r`.

## Ввод пользователя (`>?`)

Оператор `>?` читает строку из `stdin` и пытается разобрать её в целевую переменную.

```xcx
i: age;
>! "Enter age:";
>? age;
```

## Задержка (`@wait`)

Приостанавливает выполнение VM на указанное число миллисекунд.

```xcx
@wait 1000; --- Waits for 1 second
```

> `@wait` — **блокирующая** операция. Файбер при этом не отдаёт управление.

## Сырой ввод (`input`)

Модуль `input` позволяет читать отдельные клавиши без ожидания Enter.

### Методы

| Вызов               | Возврат | Описание                                    |
|---------------------|---------|---------------------------------------------|
| `input.key()`       | `s`     | Клавиша, если есть; иначе `""`              |
| `input.key() @wait` | `s`     | Ожидание нажатия клавиши                    |
| `input.ready()`     | `b`     | `true`, если в буфере есть клавиша          |

### Константы клавиш

| Значение           | Клавиша          |
|--------------------|------------------|
| `"UP"`             | Стрелка вверх    |
| `"DOWN"`           | Стрелка вниз     |
| `"LEFT"`           | Стрелка влево    |
| `"RIGHT"`          | Стрелка вправо   |
| `"ENTER"`          | Enter            |
| `"ESC"`            | Escape           |
| `"BACKSPACE"`      | Backspace        |
| `"TAB"`            | Tab              |
| `"F1"` ... `"F12"` | F1–F12           |
| `"KEY_CTRL_C"`     | Ctrl+C           |
| `"KEY_CTRL_Z"`     | Ctrl+Z           |
| `"KEY_CTRL_S"`     | Ctrl+S           |

Обычные символы возвращаются как есть: `"a"`, `"Z"`, `"5"`, `" "`.

### Пример

```xcx
s: k = input.key();
if (k == "UP") then;
    y = y - 1;
end;

--- wait for specific key
s: confirm = input.key() @wait;
if (confirm == "q") then;
    return;
end;
```

## Команды терминала (`.terminal`)

Прямое взаимодействие с системной средой или текущим процессом.

| Директива              | Описание                                      |
|------------------------|-----------------------------------------------|
| `.terminal !clear`     | Очистка экрана                                |
| `.terminal !exit`      | Завершение процесса VM                        |
| `.terminal !run s`     | Запуск другого файла XCX в новом процессе     |
| `.terminal !raw`       | Сырой режим — без эха и буферизации           |
| `.terminal !normal`    | Обычный режим терминала                       |
| `.terminal !cursor on` | Показать курсор                               |
| `.terminal !cursor off`| Скрыть курсор                                 |
| `.terminal !move x y`  | Переместить курсор в позицию `x`, `y` (`i`)   |
| `.terminal !write expr`| Вывод `expr` без перевода строки в конце      |

### Пример — игровой цикл

```xcx
.terminal !raw;
.terminal !cursor off;
.terminal !clear;

while (true) do;
    s: k = input.key();
    if (k == "ESC") then; break; end;
    if (k == "UP") then; y = y - 1; end;
    .terminal !move x y;
    .terminal !write "@";
    @wait 16;
end;

.terminal !cursor on;
.terminal !normal;
```

## Ошибки и ограничения

- **`input.key()` в обычном режиме**: возвращает `""` и выводит предупреждение `alert`.
- **`@wait` на `input.ready()`**: ошибка компиляции.
- **Директивы `.terminal`**: не являются выражениями и не могут присваиваться переменным.
- **`.terminal !move`**: аргументы должны быть целыми (`i`).
- **Доступность терминала**: при недоступных дескрипторах консоли VM завершается с `halt.fatal`.

# XCX 3.1 — обработка ошибок (Halt)

XCX использует структурированную систему `halt` для управления условиями и ошибками во время выполнения.

## Уровни halt

| Уровень      | Поведение                                              | Применение                         |
|--------------|--------------------------------------------------------|------------------------------------|
| `halt.alert` | Выводит сообщение; выполнение продолжается.            | Логирование, некритичные предупреждения |
| `halt.error` | Вывод в stderr; прерывается текущий кадр.              | Восстанавливаемые логические ошибки |
| `halt.fatal` | Вывод сообщения; немедленное завершение VM.            | Критический сбой, нарушение безопасности |

## Примеры

```xcx
halt.alert >! "Cache missed, fetching from DB...";

if (divisor == 0) then;
    halt.error >! "Division by zero!";
    return 0; --- Execution returns to caller from the recovery point
end;

if (NOT db.is_healthy()) then;
    halt.fatal >! "Database corrupted. Emergency shutdown.";
end;
```

## Семантические и runtime-паники

Некоторые недопустимые операции приводят к автоматической **панике** (аналог `halt.fatal` или `halt.error` в зависимости от контекста):
- **Деление на ноль** (арифметика)
- **Ошибка разбора JSON**: неверная строка в `json.parse()` немедленно завершает VM.
- **Обход пути**: использование `..` в методах `store` или абсолютных путей вне корня проекта вызывает `halt.fatal`.
- **Глубина рекурсии**: превышение 800 кадров вызывает `halt.error`.

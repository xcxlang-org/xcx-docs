# Лексер XCX (Scanner) — v3.1

Лексер преобразует поток исходных байтов в поток отдельных токенов.

## Детали реализации

- **Файл**: `src/lexer/scanner.rs`
- **Техника**: ручное посимвольное сканирование по `&[u8]`.
- **API**: один метод `next_token(&mut self, interner: &mut Interner) -> Token`, вызываемый парсером по требованию (не итератор).
- **Заглядывание вперёд**: один байт через `peek()`, два — через `peek_next()` / `peek_at(offset)`.

## Внутренняя структура

```rust
pub struct Scanner<'a> {
    source:   &'a [u8],   // borrowed reference to original source — no allocation
    pos:      usize,      // byte position
    char_pos: usize,      // Unicode character position (for Span.col)
    line:     usize,
    col:      usize,
}
```

`Scanner` заимствует исходник как срез байтов. Нет предварительного преобразования в `Vec<char>` — всё сканирование на уровне байтов. `char_pos` увеличивается только для байтов, не являющихся продолжением UTF-8 (`10xxxxxx`), поэтому подсчёт символов Unicode для `Span` корректен без полного декодирования.

## Типы токенов

Токены определены в `src/lexer/token.rs` как перечисление `TokenKind`. Каждый `Token` несёт `Span { line, col, len }` для отчётов об ошибках. `len` измеряется в символах Unicode (через приращения `char_pos`), а не в байтах.

Основные категории:

| Категория           | Примеры |
|---------------------|---------|
| Литералы            | `IntLiteral(i64)`, `FloatLiteral(f64)`, `StringLiteral(StringId)`, `True`, `False` |
| Ключевые слова типов| `TypeI`, `TypeF`, `TypeS`, `TypeB`, `Array`, `Set`, `Map`, `Table`, `Json`, `Date`, `Fiber` |
| Типы множеств       | `TypeSetN`, `TypeSetQ`, `TypeSetZ`, `TypeSetS`, `TypeSetB`, `TypeSetC` |
| Управление потоком  | `If`, `Then`, `ElseIf`, `Else`, `End`, `While`, `Do`, `For`, `In`, `To`, `Break`, `Continue` |
| Функции/файберы     | `Func`, `Return`, `Fiber`, `Yield` |
| Операторы           | `Plus`, `Minus`, `Star`, `Slash`, `Caret`, `PlusPlus`, `Has`, `And`, `Or`, `Not` |
| Операторы множеств  | `Union`, `Intersection`, `Difference`, `SymDifference` |
| Спец. пунктуация    | `GreaterBang` (`>!`), `GreaterQuestion` (`>?`), `DoubleColon` (`::`), `DoubleComma` (`,,`), `Bridge` (`<->`) |
| Встроенные          | `Net`, `Serve`, `Store`, `Halt`, `Terminal`, `Json`, `Date` |
| Особые              | `RawBlock(StringId)`, `AtStep`, `AtAuto`, `AtWait` |

## Особенности сканирования

### Сырые блоки (Raw Blocks)
Ограничены `<<<` и `>>>`. Всё между ними — один токен `RawBlock(StringId)` для встроенного JSON или многострочных данных.

```
<<<
  { "key": "value" }
>>>
```

Обнаружение: `self.source[self.pos..].starts_with(b"<<<")`. Содержимое читается побайтно до `>>>`.

### Комментарии
В XCX разделитель комментариев — `---`. Обнаружение: `starts_with(b"---")`:
- **Однострочный**: `--- this is a comment` (непустое содержимое после `---` на той же строке)
- **Многострочный**: `---` и только пробелы до конца строки открывают блок, закрытие — `*---`

Закрытие многострочного: `starts_with(b"*---")`. После `---` сканер смотрит вперёд, чтобы выбрать режим.

### Unicode-операторы множеств
Символы распознаются по UTF-8 последовательностям:
- `∪` → `TokenKind::Union`
- `∩` → `TokenKind::Intersection`
- `\` (ASCII backslash) → `TokenKind::Difference`
- `⊕` → `TokenKind::SymDifference`

Для многобайтовых операторов `advance()` вызывается дополнительно для оставшихся байт продолжения (`c >= 128`).

### Различение else/elseif
После `else` / `els` сканер смотрит, не следует ли `if` — тогда оба слова сливаются в один токен `ElseIf`. Сохранённая позиция позволяет откатиться, если следующее слово не `if`.

### Директивы `@`
Последовательности после `@` читаются как ASCII-буквы и сопоставляются:
- `@step` → `AtStep`
- `@auto` → `AtAuto`
- `@wait` → `AtWait`

Неизвестные последовательности дают `TokenKind::Unknown('@')`.

### Две точки (`..`) → `To`
Две точки `..` — токен `To` (диапазоны), в отличие от одной `.` (`Dot`).

### Двойная запятая (`,,`) → `DoubleComma`
`,,` — `DoubleComma` (литералы диапазонов множеств: `set:N { 1,,10 }`).

### Сканирование идентификаторов
`identifier()` собирает подряд ASCII-алфанумерику, подчёркивания и байты `>= 128`. Диапазон байтов преобразуется через `from_utf8` и приводится к нижнему регистру для ключевых слов. Сначала проверяются регистрозависимые совпадения (`"N"`, `"UNION"` и т.д.). Остальное — через `Interner::intern()`.

### Сканирование чисел
`number()` накапливает цифры. Если после `.` идёт цифра — `FloatLiteral`. Разбор через `str::from_utf8` и `.parse()`.

### Сканирование строк
`string()` обрабатывает escape (`\n`, `\t`, `\r`, `\"`, `\\`, `\NNN`, `\xHH`), строит `Vec<u8>`, затем `String::from_utf8` и интернирование.

## Выделение строк

Все идентификаторы и строковые литералы проходят через `Interner::intern()` → `StringId (u32)`. Далее по конвейеру используются числовые ID без повторных сравнений строк в куче.

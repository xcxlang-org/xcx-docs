# Руководство по пакетному менеджеру PAX

PAX — официальный пакетный менеджер XCX, встроенный непосредственно в бинарный файл `xcx`. Он управляет зависимостями, scaffolding'ом проектов, публикацией в реестр и автоматизацией сборки.

---

## Конфигурация проекта: `project.pax`

Каждый PAX-проект должен содержать файл `project.pax` в корневом каталоге. Используется пользовательский декларативный формат.

```pax
---
PAX Project Configuration
*---
/
    name        :: "my_project",
    version     :: "1.0.0",
    author      :: "yourname",
    description :: "A short description of the project",
    main        :: "src/main.xcx",
    tags  :: ["tag1", "tag2"],
    files :: ["src/main.xcx", "src/lib.xcx"],
    deps  :: [
        "somelib@1.0.0",
        "author/repo@latest",
        "https://example.com/lib.xcx"
    ]
/
```

### Поля

| Поле          | Обязательное | Описание                                                                     |
|---------------|--------------|------------------------------------------------------------------------------|
| `name`        | Да           | Логическое имя проекта/пакета.                                               |
| `version`     | Да           | Строка семантической версии (например, `"1.0.0"`).                           |
| `author`      | Нет          | Имя автора, отображаемое в реестре.                                          |
| `description` | Нет          | Краткое описание проекта.                                                    |
| `main`        | Нет          | Пользовательская точка входа. По умолчанию `src/main.xcx`, если не указана. |
| `tags`        | Нет          | Список тегов для обнаружения в реестре.                                      |
| `files`       | Нет          | Явный список файлов для включения при публикации. Если не указан, PAX автоматически исключает некодовые файлы. |
| `deps`        | Нет          | Список зависимостей. Поддерживает версионированные имена, сокращения `author/repo` и прямые URL. |

### Форматы зависимостей

```pax
deps :: [
    "mylib@1.0.0",           --- пакет из реестра с конкретной версией
    "mylib@latest",          --- последняя версия из реестра
    "author/mylib@2.1.0",    --- с префиксом автора
    "https://example.com/lib.xcx"  --- прямой URL
]
```

## Файл блокировки: `pax.lock`

PAX автоматически ведёт файл `pax.lock` после каждой установки. В нём записываются точная разрешённая версия и локальный путь каждой установленной зависимости.

```json
{
  "mylib": { "version": "mylib@1.0.0", "file": "lib/mylib/" },
  "otherlib": { "version": "otherlib@latest", "file": "lib/otherlib/" }
}
```

- **Не редактируйте** `pax.lock` вручную.
- Фиксируйте его в системе контроля версий для воспроизводимых сборок.

## Структура каталогов

Стандартный PAX-проект имеет следующую структуру:

```
my_project/
├── project.pax       # Конфигурация проекта
├── pax.lock          # Автоматически генерируемый файл блокировки
├── src/
│   └── main.xcx      # Главная точка входа
├── lib/              # Установленные зависимости (управляется PAX)
└── tests/            # Тесты проекта
```

## Справочник команд

PAX вызывается через `xcx pax <command>`.

| Команда                      | Описание                                               |
|------------------------------|--------------------------------------------------------|
| `xcx pax new <name>`         | Создать новую структуру проекта.                       |
| `xcx pax install`            | Загрузить все зависимости в `lib/`.                    |
| `xcx pax add <dep>`          | Добавить зависимость и сразу установить её.            |
| `xcx pax remove <name>`      | Удалить зависимость из `project.pax`.                  |
| `xcx pax clone <package>`    | Загрузить опубликованный пакет как локальный проект.   |
| `xcx pax run [path]`         | Выполнить точку входа проекта.                         |
| `xcx pax search <query>`     | Найти доступные пакеты в реестре.                      |
| `xcx pax login <token>`      | Сохранить токен аутентификации реестра.                |
| `xcx pax logout`             | Удалить сохранённый токен аутентификации.              |
| `xcx pax whoami`             | Проверить текущую учётную запись реестра.              |
| `xcx pax publish`            | Опубликовать проект в реестре PAX.                     |

---

## Подробное описание команд

### `xcx pax new <name>`

Создаёт новый проект со стандартной структурой каталогов.

```sh
xcx pax new my_project
```

Создаёт:
```
my_project/
├── project.pax
├── src/
│   └── main.xcx
└── README.md
```

---

### `xcx pax install`

Читает `project.pax`, разрешает все `deps` и загружает их в `lib/`. Пропускает уже присутствующие пакеты (по имени).

```sh
xcx pax install
```

- Устанавливает только файлы, объявленные в поле `files` самой зависимости.
- Если у зависимости нет поля `files`, PAX автоматически исключает `README.md`, `LICENSE`, `tests/`, `docs/`, `.git*` и `project.pax`.
- Обновляет `pax.lock` после каждой успешной загрузки.

---

### `xcx pax add <dep>`

Добавляет зависимость в `project.pax` и сразу запускает установку.

```sh
xcx pax add mylib@1.0.0
xcx pax add author/mylib@latest
```

---

### `xcx pax remove <name>`

Удаляет запись зависимости из `project.pax`.

```sh
xcx pax remove mylib
```

> **Примечание:** Папки `lib/` и записи `pax.lock` не очищаются автоматически. Повторно запустите `xcx pax install` или удалите их вручную.

---

### `xcx pax clone <package>`

Загружает опубликованный пакет из реестра PAX в виде полноценного локального проекта, готового к запуску или изменению.

```sh
xcx pax clone snake_game
xcx pax clone author/snake_game
```

- Создаёт каталог с именем пакета в текущем рабочем каталоге.
- Копирует полную структуру проекта, включая `project.pax`, `src/` и все объявленные файлы.
- **Не** устанавливает зависимости автоматически.

**Типичный рабочий процесс:**
```sh
xcx pax clone snake_game
cd snake_game
xcx pax install
xcx pax run
```

---

### `xcx pax run [path]`

Выполняет проект. Порядок разрешения точки входа:

1. Путь, переданный как аргумент (например, `xcx pax run other/main.xcx`)
2. Поле `main` в `project.pax`
3. По умолчанию: `src/main.xcx`

```sh
xcx pax run
xcx pax run src/alt.xcx
```

---

### `xcx pax search <query>`

Ищет в реестре PAX пакеты, соответствующие запросу.

```sh
xcx pax search json
```

Формат вывода:
```
- json_utils (v1.2.0) by alice
- fast_json (v0.9.1) by bob
```

---

### `xcx pax login <token>`

Сохраняет токен аутентификации реестра локально в `.pax_token`.

```sh
xcx pax login your_token_here
```

---

### `xcx pax logout`

Удаляет сохранённый токен аутентификации.

```sh
xcx pax logout
```

---

### `xcx pax whoami`

Проверяет текущую сессию реестра и выводит информацию об учётной записи.

```sh
xcx pax whoami
# Account: alice [developer]
```

---

### `xcx pax publish`

Публикует текущий проект в реестре PAX. Требуется вход в систему.

```sh
xcx pax publish
```

- Читает метаданные из `project.pax` (`name`, `version`, `description`).
- Сканирует и загружает все файлы проекта (исключая `.zip`, `.pax_token`, `.git`).
- Отправляет сжатый архив и манифест файлов в реестр.

> **Примечание:** Обновляйте `version` в `project.pax` перед каждой публикацией. Реестр может отклонить дублирующиеся версии.

---

## Конфигурация реестра

По умолчанию PAX подключается к `pax.xcxlang.com`. Это можно переопределить через файл `.pax_config` в корне проекта:

```json
{
  "registry": "https://my-custom-registry.example.com"
}
```

- HTTP используется автоматически для `localhost` и `127.0.0.1`.
- HTTPS используется для всех остальных хостов.
